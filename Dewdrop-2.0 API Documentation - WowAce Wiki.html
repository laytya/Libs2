Standard methods

[edit]

 :Register(parent, ...)

Registers a given parent with Dewdrop-2.0. Settings are initialized from the other arguments given.

[edit]

Args

parent 
Frame - parent frame to register.
... 
#Register/Open arguments
[edit]

Remarks

This automatically registers the right-click with with the parent frame. Unregistering will cause this not to happen.

[edit]

Example 

dewdrop:Register(Minimap,
    'children', function()
        dewdrop:AddLine('text', "Text")
    end
)
[edit]

 :Unregister(parent)

Unregisters a given parent from Dewdrop-2.0.

[edit]

Args

parent 
Frame - registered parent frame.
[edit]

Example 

dewdrop:Unregister(Minimap)
[edit]

 :Open(parent, ...) or (parent, otherParent) or (parent)

Opens the menu on the given parent, the innards are initialized either with data from Register(parent, ...) or from the given arguments.

[edit]

Args

parent 
Frame - parent frame to open.
... 
#Register/Open arguments
otherParent 
Frame - parent frame to get information on.
[edit]

Remarks

If you called Register(parent, ...) on the same parent, you do not need to give extra arguments, but if you do, they will take precedent.

[edit]

Example 

dewdrop:Open(Minimap) -- assuming data from the registry
[edit]

 :Close([level])

Closes the given level and sublevels

[edit]

Args

level 
number [1, inf) - level to close. (default: 1)
[edit]

Example 

dewdrop:Close(2) -- closes 2nd levels and any sublevels
[edit]

 :Refresh([level])

Refreshes the contents of a given level

[edit]

Args

level 
number [1, inf) - level to refresh. (default: 1)
[edit]

Example 

dewdrop:Refresh(2) -- refreshes the second level.
[edit]

 :AddLine(...)

Adds a line to the dropdown menu.

[edit]

Args

... 
A list of key-value pairs to represent information. #AddLine arguments
[edit]

Remarks

This should only be called inside the init function of Open(frame, func).

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, World!"
)
[edit]

 :IsOpen(parent)

Returns whether the given parent frame has an open dropdown menu on it.

[edit]

Args

parent 
Frame - parent frame to check.
[edit]

Returns

boolean - whether the given parent frame has an open dropdown menu.

[edit]

Example 

local open = dewdrop:IsOpen(Minimap)
[edit]

 :IsRegistered(parent)

Returns whether the given parent frame has been registered with Dewdrop.

[edit]

Args

parent 
Frame - parent frame to check.
[edit]

Returns

boolean - whether the given parent frame has been registered with Dewdrop.

[edit]

Example 

local registered = dewdrop:IsRegistered(Minimap)
[edit]

Register/Open arguments

[edit]

children

function(level, value, valueN_1, valueN_2, valueN_3, valueN_4) 
function to describe the innards of the menu.
table - An AceOptions_data_table
[edit]

Remarks

value is the value of the menu just opened, where the level number is N.
valueN_1 is the value of level N-1, and so forth onto valueN_4, which is the value of level N-4.

[edit]

Example 

dewdrop:Register(Minimap, 
    'children', function(level, value)
        dewdrop:AddLine('text', "Text")
    end
)
[edit]

point

string 
the main attach point of the menu.
function 
a function which returns the main attach point of the menu and, optionally, the relative attach point.
[edit]

Example 

dewdrop:Register(Minimap,
    'point', function(parent)
        return "TOPRIGHT", "TOPLEFT"
    end
)
[edit]

relativePoint

string 
the relative attach point of the menu.
function 
a function which returns the relative attach point of the menu.
[edit]

Example 

dewdrop:Register(Minimap,
    'point', function(parent)
        return "TOPRIGHT"
    end,
    'relativePoint', function(parent)
        return "TOPLEFT"
    end
)
[edit]

cursorX

boolean - Whether the menu follows the cursor on the x-axis

[edit]

Remarks

In this case, point should be "BOTTOM" or "TOP", and relativePoint should be the opposite.

[edit]

Example 

dewdrop:Register(Minimap,
    'point', "TOP",
    'relativePoint', "BOTTOM",
    'cursorX', true
)
[edit]

cursorY

boolean - Whether the menu follows the cursor on the y-axis

[edit]

Remarks

In this case, point should be "LEFT" or "RIGHT", and relativePoint should be the opposite.

[edit]

Example 

dewdrop:Register(Minimap,
    'point', "LEFT",
    'relativePoint', "RIGHT",
    'cursorY', true
)
[edit]

dontHook [Register only]

boolean - whether the mouse click scripts will be hooked to open/close the menu. You will need to manually call :Open() if you do this.

[edit]

Example 

dewdrop:Register(Minimap,
    'point', "LEFT",
    'dontHook', true
)
[edit]

AddLine arguments

[edit]

text

string - the text on the button.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, World!",
)
[edit]

level

number - the level the line should be added at.

[edit]

Remarks

In almost all cases, you won't need this, as the level is inferred automatically.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, World!",
    'level', 2
)
[edit]

disabled

boolean - whether the button is disabled.

[edit]

Remarks

Disabling causes the button to be unclickable and makes the text gray.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, World!",
    'disabled', true
)
[edit]

isTitle

boolean - whether the button is a title.

[edit]

Remarks

Being a title causes the button to be disabled, but have the standard yellow color.

[edit]

Example 

dewdrop:AddLine(
    'text', "I am a title",
    'isTitle', true
)
[edit]

notClickable

boolean - whether the button is clickable or not.

[edit]

Remarks

Being unclickable causes the button to be disabled, but have the standard white color, so it would look like other buttons.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'notClickable', true
)
[edit]

textR, textG, textB

textR 
number [0, 1] - red value
textG 
number [0, 1] - green value
textB 
number [0, 1] - blue value
[edit]

Remarks

Sets the text color of the button, overriding disabled, isTitle, or notClickable's colors.
This only works if all three colors are set.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'textR', 1,
    'textG', 1,
    'textB', 1
)
[edit]

notCheckable

boolean - whether the button is checkable or not.

[edit]

Remarks

Really, this being set just causes the blank space that would be normally for a checkbox to not be there.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'notCheckable', true
)
[edit]

isRadio

boolean - whether the button is a radio button.

[edit]

Remarks

If set to true, it uses a texture similar to "Interface\Buttons\UI-RadioButton", which is 4 buttons side-by-side, unchecked, checked, highlight, and blank.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'isRadio', true,
)
[edit]

checked

boolean - whether the button is checked or not.

[edit]

Remarks

If set to true, it sets the check's alpha to fully visible, otherwise makes it invisible.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'checked', math.random() < 0.5
)
[edit]

checkIcon

string - path to the check texture.

[edit]

Remarks

If not set, assume "Interface\\Buttons\\UI-CheckBox-Check"

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'checkIcon', "Interface\\Buttons\\UI-CheckBox-Check"
)
[edit]

func

function(arg1, arg2, arg3, arg4) - function to call when button is clicked.

[edit]

Remarks

The args fed in are the ones given by arg1, arg2, arg3, and arg4.
After being called, the menu level is refreshed, so any checks or whatnot are updated.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'func', function()
        print("Click!")
    end
)
[edit]

arg1, arg2, arg3, arg4

arg1 
Value - First value fed into the given func
arg2 
Value - Second value fed into the given func
arg3 
Value - Third value fed into the given func
arg4 
Value - Fourth value fed into the given func
[edit]

Remarks

You don't need all 4 to work, you may only specify if needed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'func', function(val)
        print("Click! " .. val)
    end,
    'arg1', math.random()
)
[edit]

hasColorSwatch

boolean - whether the button has a color swatch.

[edit]

Remarks

If hasColorSwatch is true, then func has no purpose, since clicking opens up a color picker.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasColorSwatch', true
)
[edit]

r, g, b

r 
number [0,1] - The red value.
g 
number [0,1] - The green value.
b 
Number [0,1] - The blue value.
[edit]

Remarks

Sets the color of the color swatch, and conseqently, the color picker.
This only works if all three colors are set.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasColorSwatch', true,
    'r', 1,
    'g', 1,
    'b', 1
)
[edit]

colorFunc

function(r, g, b [, a]) - function called when the color changes.

[edit]

Remarks

Requires that hasColorSwatch is true.
The [a] argument is provided if hasOpacity is true.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasColorSwatch', true,
    'hasOpacity', true,
    'colorFunc', function(r, g, b, a)
        print("New color: " .. r .. "-" .. g .. "-" .. b .. '-' .. a)
    end
)
[edit]

colorArg1, colorArg2, colorArg3, colorArg4

colorArg1 
Value - First value fed into the given colorFunc
colorArg2 
Value - Second value fed into the given colorFunc
colorArg3 
Value - Third value fed into the given colorFunc
colorArg4 
Value - Fourth value fed into the given colorFunc
[edit]

Remarks

You don't need all 4 to work, you may only specify if needed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'colorFunc', function(value, r, g, b, a)
        print("New color: " .. value .. " :: " .. r .. "-" .. g .. "-" .. b .. '-' .. a)
    end,
    'colorArg1', math.random()
)
[edit]

opacity

number [0,1] - current opacity level.

[edit]

Remarks

Requires that hasColorSwatch and hasOpacity are true.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasColorSwatch', true,
    'hasOpacity', true
    'opacity', math.random()
)
[edit]

hasArrow

boolean - whether button has an arrow.

[edit]

Remarks

Requires that either value is not nil or hasSlider is true.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'value', "hello"
)
[edit]

value

value - any unique, identifiable value.

[edit]

Remarks

Requires that hasArrow is true.
The same value will be fed in to the initalizer function when the sublevel is opened.

[edit]

Example 

if level == 1 then
    dewdrop:AddLine(
        'text', "Hello,",
        'hasArrow', true,
        'value', "hello"
    )
elseif level == 2
    if value == "hello" then
        dewdrop:AddLine(
            'text', "World!"
        )
    end
end
[edit]

hasSlider

boolean - whether the sublevel is a slider.

[edit]

Remarks

Requires that hasArrow is true.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
)
[edit]

sliderMin

number - the minimum value

[edit]

Remarks:

Requires that hasArrow and hasSlider are true.
If not given, 0 is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderMin', 0
)
[edit]

sliderMax

number - the minimum value

[edit]

Remarks:

Requires that hasArrow and hasSlider are true.
If not given, 1 is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderMax', 1
)
[edit]

sliderIsPercent

boolean - whether sliderMinText and sliderMaxText will be percents.

[edit]

Remarks:

Requires that hasArrow and hasSlider are true.
If not given, false is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderIsPercent', true
)
[edit]

sliderMaxText

string - the text representing the max value

[edit]

Remarks:

Requires that hasArrow and hasSlider are true.
If not given, 100 * sliderMax .. "%" is assumed if a percentage, otherwise, just sliderMax.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderMaxText', '100%'
)
[edit]

sliderMinText

string - the text representing the min value

[edit]

Remarks:

Requires that hasArrow and hasSlider are true.
If not given, 100 * sliderMin .. "%" is assumed if a percentage, otherwise, just sliderMin.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderMinText', '0%'
)
[edit]

sliderValue

number [0,1] - the current value of the slider.

[edit]

Remarks

Requires that hasArrow and hasSlider are true. If not given, 0.5 is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderValue', 0.5
)
[edit]

sliderFunc

function(value) - function to call when the value changed.

[edit]

Remarks

Requires that hasArrow and hasSlider are true. If sliderFunc(value) returns a string, then that is used to set the current value string on the slider. value will always be within [0, 1].

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderFunc', function(value)
        print("New value: " .. value)
        return format("%.0f%%", value)
    end
)
[edit]

sliderArg1, sliderArg2, sliderArg3, sliderArg4

sliderArg1 
Value - First value fed into the given sliderFunc
sliderArg2 
Value - Second value fed into the given sliderFunc
sliderArg3 
Value - Third value fed into the given sliderFunc
sliderArg4 
Value - Fourth value fed into the given sliderFunc
[edit]

Remarks

You don't need all 4 to work, you may only specify if needed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasSlider', true
    'sliderFunc', function(key, value)
        print("New " .. key .. ": " .. value)
        return format("%.0f%%", value)
    end,
    'sliderArg1', "hello"
)
[edit]

textHeight

number - height of the text.

[edit]

Remarks

If not provided, 10 is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'textHeight', 10
)
[edit]

justifyH

string - horizontal justification of the text.

[edit]

Remarks

Must be either "LEFT", "RIGHT", or "CENTER"
If not provided, "LEFT" is assumed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'justifyH', "CENTER"
)
[edit]

tooltipTitle

string - title of the newbie tooltip.

[edit]

Remarks

If not provided, the tooltip won't show up.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'tooltipTitle', "Hello"
)
[edit]

tooltipText

string - text of the newbie tooltip.

[edit]

Remarks

Requires tooltipTitle to be set.

[edit]

Example

    'text', "Hello, world!",
    'tooltipTitle', "Hello",
    'tooltipText', "world!"
)
[edit]

tooltipFunc

function - A function to call when the tooltip is to be shown.

[edit]

Remarks

Will set the owner and location of GameTooltip.

[edit]

Example

    'text', "Hello, world!",
    'tooltipFunc', GameTooltip.SetHyperlink,
    'tooltipArg1', GameTooltip,
    'tooltipArg2', "item:1234:0:0:0"
)
[edit]

tooltipArg1, tooltipArg2, tooltipArg3, tooltipArg4

tooltipArg1 
Value - First value fed into the given tooltipFunc
tooltipArg2 
Value - Second value fed into the given tooltipFunc
tooltipArg3 
Value - Third value fed into the given tooltipFunc

Value - Fourth value fed into the given tooltipFunc

[edit]

Remarks

[edit]

[edit]

editBoxFunc

function(text) - function to call when the enter key is pressed.

[edit]

Remarks

Requires that hasArrow and hasEditBox are true.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasEditBox', true,
    'editBoxFunc', function(text)
        print("New text: " .. text)
    end
)
[edit]

editBoxArg1, editBoxArg2, editBoxArg3, editBoxArg4

editBoxArg1 
Value - First value fed into the given editBoxFunc
editBoxArg2 
Value - Second value fed into the given editBoxFunc
editBoxArg3 
Value - Third value fed into the given editBoxFunc
editBoxArg4 
Value - Fourth value fed into the given editBoxFunc
[edit]

Remarks

You don't need all 4 to work, you may only specify if needed.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasEditBox', true,
    'editBoxFunc', function(key, text)
        print("New " .. key .. ": " .. text)
    end,
    'editBoxArg1', "hello"
)
[edit]

editBoxChangeFunc

function(text) - function to call when the text changes.

[edit]

Remarks

Requires that hasArrow and hasEditBox are true.
If this returns a string, the text box will be changed to it.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasEditBox', true,
    'editBoxChangeFunc', function(text)
        value = string.upper(text)
        print("New value: " .. text)
        return value -- changes text to uppercase
    end
)

editBoxValidateFunc

function(text) - function to call when the enter key is pressed.

[edit]

Remarks

Requires that hasArrow and hasEditBox are true.
If this returns a true value, it will pass through, if it returns a false value, a message will pop up.

[edit]

Example 

dewdrop:AddLine(
    'text', "Hello, world!",
    'hasArrow', true,
    'hasEditBox', true,
    'editBoxValidateFunc', function(text)
        -- no spaces
        return not string.find(text, "%s")
    end
)

editBoxValidateArg4 
Value - Fourth value fed into the given editBoxValidateFunc
